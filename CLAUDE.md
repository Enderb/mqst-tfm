## LaTeX class constraints (`quantumarticle.cls`)

- **No `\\` in `\title{}`**. The class saves the title in a single-line `\savebox` to decide font size. Inside that box `\raggedright` redefines `\\` to `\@centercr`, which calls `\par` and corrupts list state → "Something's wrong--perhaps a missing \item". Use a long title on one line; the class wraps it automatically in the displayed parbox.
- The class blacklists `lmodern` and several font packages — it loads them itself. Adding them again causes package conflicts.
- Abstract text and `\maketitle` are co-dependent: the class calls `\@maketitle` inside the `abstract` environment. In `main.tex` the abstract environment is redefined (`\renewenvironment{abstract}`) to avoid this, since the title page is a manual `titlepage` environment, not `\maketitle`.
- Max 20 pages of main text (appendices do not count). As of the July 2026 review the main text is **exactly 20 pages** — there is no slack. Any added float/paragraph must be paid for by a cut elsewhere. **Page-count check:** the bibliography continues arabic numbering (page 21+), so do NOT read its page numbers as main text. Measure from the Introduction *body* page to the first Bibliography *content* page:
  ```bash
  intro=$(for p in $(seq 1 40); do pdftotext -f $p -l $p main.pdf - 2>/dev/null | grep -q "Quantum computing is expected" && echo $p && break; done)
  biblio=$(for p in $(seq 1 40); do t=$(pdftotext -f $p -l $p main.pdf - 2>/dev/null); echo "$t"|grep -q "^Bibliography" && echo "$t"|grep -qE "Stefanazzi|Krantz" && echo $p && break; done)
  echo "MAIN TEXT PAGES: $((biblio - intro))"
  ```

| Metric | LDA | 1D CNN | Gain |
|---|---|---|---|
| Overall fidelity | 0.826 | **0.9975** | +17.1 pp |
| \|0⟩ acc | 0.875 | 1.000 | +12.5 pp |
| \|1⟩ acc (all) | 0.778 | 0.995 | +21.8 pp |
| \|1⟩ acc (relaxation only) | **0.443** | **0.984** | **+54.1 pp** |

CNN = 953 params, `EPOCHS=60`, 2000 shots/state. **0.9975 is the canonical headline overall fidelity** (user-confirmed; the epoch-40 value 0.9950 also appears in the notebook). The LDA overall here (0.826) is *lower* than Part I's 0.906 because the synthetic set deliberately over-represents relaxation.

Synthetic T1: `T1_samples = 600` (≈3.0 µs at 200 MSps), window T=200 samples, τ_cav=κ⁻¹=20 samples → relaxation fraction **≈ 28.3 %** (~566/2000 shots; 122 in the 800-shot test set). This is intentionally *higher* than the real device — a stress test of the recovery. To model the real device, recalibrate T1 to ~1200–1500 samples to match the measured ~12.75 % \|1⟩ error rate (future work). Do **not** claim the synthetic gap is the expected real-device gain; the robust claim is the qualitative raw-trace advantage.

**Framing of "synthetic" (supervisor-corrected, do not regress):** the raw traces are synthetic because the QPU was not accessible in time — Parts I/II used real *integrated* IQ from Qilimanjaro's databases, and raw per-sample traces are not routinely stored, so none existed to train the CNN. Collecting them is a motivation for setting up the ZCU216 (`acquire_decimated`). "Synthetic" = built from the real device's IQ statistics (centroids/covariances), not invented. Do NOT frame the missing raw data as an `acquire` vs `acquire_decimated` choice — that was an earlier wrong framing; the real reason is chip unavailability. Eq. 12 (cavity-response trace model) cites `Blais2021,Gambetta2007`.

## Dátil resonator frequencies

6.5051 GHz, 6.7304 GHz, 6.9781 GHz, 7.3892 GHz. Minimum adjacent separation = **225 MHz** (6.7304 − 6.5051); this is the number quoted in ch. 05 for PFB channel spacing (corrected from an earlier 184 MHz). All four lie above the XM655 BALUN passband (≤ 6 GHz), so multiplexed readout needs direct HD-header connections with discrete BALUNs.

## Key physical constants and calibration values

- Phase model: `φ(f) = φ₀ − 360·τ·f`; τ attributed to DAC/ADC pipeline latency (not loopback cable length — prior claim was an error, corrected in ch. 04)
- ZCU216 converters: DAC up to 9.85 GS/s, ADC up to 2.5 GS/s, both **14-bit** (the ADC is *not* 12-bit — corrected in ch. 03; the part is the Gen-3 XCZU49DR)
- Accumulated readout noise: σ ≈ 1.9 ADU / √N_reps
- tProcessor fabric clock: 384 MHz → 1 cycle ≈ 2.6 ns
- Active reset digital latency: `condj` + threshold ≈ 10–13 ns total
- Channel mapping: Generator 6 = DAC 2_231 (JHC3); Readout 0 = ADC 0_226 (JHC5)
- κ symbol: in notebooks, the variable currently named `kappa_samples` should be renamed `tau_cav_samples` to avoid clash with the κ linewidth symbol used in the text (1/κ = τ_cav)

## Voice / style conventions (enforced in the July 2026 review)

The thesis must read as a careful MSc student, not AI-generated. When editing prose:
- No em-dashes as a stylistic default — use commas, colons, or restructure. (Empty `minted`/table cells `& —` are fine.)
- No hedging/filler openers ("it is worth noting", "notably", "furthermore", "crucially").
- No generated transitions ("Building on this", "Having established X, we now…").
- Avoid hype adjectives (compelling, powerful, seamless, striking) unless technically precise.
- Vary sentence rhythm; avoid mechanical "not only X but also Y" and rule-of-three lists.
Quick grep before committing prose edits: `grep -rniE "it is worth noting|notably,|furthermore,|building on|compelling|not only" chapters/`.
